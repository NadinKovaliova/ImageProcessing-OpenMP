# Контрольна робота з дисципліни «Паралельне та розподілене моделювання складних інформаційних систем».

## Контрольна робота: Паралельна обробка зображень з використанням OpenMP

## Анотація

Проєкт демонструє застосування технології OpenMP для розпаралелювання обчислювально інтенсивних завдань на прикладі цифрової обробки зображень. Програмно реалізовано алгоритм перетворення повноколірного растрового зображення (формату BMP) у відтінки сірого (grayscale). 

Ключовою задачею є кількісна оцінка та порівняльний аналіз продуктивності двох реалізацій одного й того ж алгоритму: послідовної (однопоточної) та паралельної (багатопоточної). Ефективність розпаралелювання оцінюється шляхом вимірювання часу виконання обох версій та розрахунку коефіцієнта прискорення.

## Використані технології та засоби розробки

*   **Мова програмування:** C++ 
*   **Технологія паралелізму:** OpenMP API
*   **Середовище розробки:** Microsoft Visual Studio 2022 (компілятор MSVC)
*   **Стандартні бібліотеки:** `iostream`, `vector`, `chrono`, `fstream`, `stdexcept`
*   **Системні бібліотеки:** `Windows.h` (для функцій консолі)

## Інструкція з компіляції та запуску

Для відтворення результатів необхідно виконати наступну послідовність дій:

1.  **Клонування репозиторію:**
    Завантажити вихідний код проєкту на локальну машину за допомогою команди:
    ```bash
    git clone https://github.com/NadinKovaliova/ImageProcessing-OpenMP.git
    ```

2.  **Відкриття проєкту в середовищі розробки:**
    Відкрити файл рішення `контрольна робота.sln` за допомогою Microsoft Visual Studio 2022 або новішої версії.

3.  **Налаштування параметрів компілятора:**
    Необхідно активувати підтримку OpenMP. Для цього у властивостях проєкту (`Project -> Properties`) слід перейти до розділу `Configuration Properties -> C/C++ -> Language` та встановити для параметра `Open MP Support` значення `Yes (/openmp)`.

4.  **Підготовка вхідних даних:**
    У головну директорію проєкту (де знаходиться файл `контрольна робота.cpp`) необхідно помістити вхідне зображення. Файл повинен відповідати наступним вимогам:
    *   Назва файлу: **`input.bmp`**
    *   Формат: **24-розрядний рисунок (*.bmp)** без стиснення.
    *   Для наочної демонстрації ефекту прискорення рекомендується використовувати зображення великої роздільної здатності (наприклад, 1920x1080 пікселів або більше).

5.  **Запуск виконання:**
    Скомпілювати та запустити проєкт у режимі без налагодження, натиснувши комбінацію клавіш `Ctrl + F5`.

## Очікувана поведінка та результати роботи

При запуску програма послідовно виконує наступні дії:

1.  **Етап 1: Послідовна обробка.**
    *   Завантажує дані з файлу `input.bmp`.
    *   Виконує алгоритм перетворення у відтінки сірого в одному потоці.
    *   Виводить у консоль час, витрачений на виконання.
    *   Зберігає результат у файл **`output_sequential.bmp`**.

2.  **Етап 2: Паралельна обробка.**
    *   Повторно завантажує вихідні дані з `input.bmp`.
    *   Виконує той самий алгоритм, але з використанням директиви `#pragma omp parallel for`, що розподіляє обчислення між усіма доступними потоками процесора.
    *   Виводить у консоль час, витрачений на паралельне виконання.
    *   Зберігає результат у файл **`output_parallel.bmp`**.

3.  **Етап 3: Підсумок.**
    *   У консоль виводиться зведена інформація, що містить кількість потоків, задіяних OpenMP, та розрахований коефіцієнт прискорення (відношення часу послідовного виконання до паралельного).

Внаслідок роботи програми у директорії проєкту будуть створені два файли-артефакти, що є чорно-білими версіями вихідного зображення.
